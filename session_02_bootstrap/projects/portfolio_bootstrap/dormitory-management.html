System Analysis, Modeling & Database Design

Dormitory
Management System

The Dormitory Management System is a university team project developed for the Analysis and Design of Information Systems course. The project aims to build an efficient system that streamlines student profiling, room allocations, and contract management within the university campus.

In this project, I took full ownership of the system analysis, modeling, and database design phases. My primary responsibilities included:

  • System Modeling & Diagramming: Designing the overview Use Case diagram, detailed Use Case decomposition for student management, Activity diagrams (specifically for student management and profile viewing), and Sequence diagrams to map out the system's runtime behavior.
  • UI & Database Design: Constructing the User Interface (UI) flow diagrams for the student management module and designing the structural database schema.
  • Documentation: Collaborating with the team to write, refine, and compile the comprehensive final project report.

Through this project, I significantly strengthened my logical thinking, system modeling, and technical documentation skills. This practical experience provided me with deep insights into the software analysis and design lifecycle, reinforcing my foundational skills as a future software engineer.
